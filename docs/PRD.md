# 打工魂·不灭.skill — 产品需求文档（PRD）

---

## 一、产品概述

**打工魂·不灭.skill** 是一个运行在 Claude Code 上的 meta-skill。

用户通过对话式交互提供原材料（工作记录、聊天截图、考勤数据、邮件等），系统自动分析并生成一个可独立运行的**案件 Skill**。

- 用户可以针对不同公司创建多个案件 Skill，互不干扰
- 生成的 Skill 内置核心劳动法知识，结合个人案情给出针对性建议

### 核心理念

**把你在某家公司试用期期间的所有信息——工作记录、打卡、聊天、邮件、工作成果——蒸馏成一个结构化的"案件 Skill"**，它既是你的证据库，也是你的维权顾问。

### 与参考项目的关系

| 维度 | 同事.skill | 前任.skill | 打工魂·不灭.skill |
|------|-----------|-----------|-----------------|
| 蒸馏对象 | 同事 | 前任 | **你自己**（某家公司的那个阶段） |
| Part A | Work Skill | Relationship Memory | **案情档案**（合同、时间线、证据链） |
| Part B | Persona | Persona | **维权策略**（法律分析、应对方案、文书模板） |
| 产出目标 | 模拟同事对话 | 模拟前任对话 | **仲裁证据包 + 维权顾问** |
| 多实例 | 多个同事 | 多个前任 | **多家公司** |

## 三、用户流程

```
用户触发 /create-dagong-soul
        ↓
[Step 1] 基础信息录入（全部可跳过）
  - 公司名称
  - 入职日期 / 试用期起止
  - 岗位和职级
  - 合同约定薪资（月薪/年薪）
  - 试用期约定时长
  - 被通知不转正的时间节点
  - 公司给出的理由
  - 你认为公司理由是否合理（主观判断）
  - 目前进展（约谈中/已收到通知/已在仲裁）
        ↓
[Step 2] 原材料导入（可跳过，后续追加）
  - 聊天记录（微信/钉钉/飞书截图或导出）
  - 邮件（.eml / 截图）
  - 劳动合同（照片/PDF）
  - 考勤记录（截图/导出）
  - 绩效评估（截图/文档）
  - 工作成果（代码截图、文档、项目记录）
  - HR/领导约谈记录（截图/文字描述）
  - 公司规章制度（员工手册等）
  - 工资条/银行流水
        ↓
[Step 3] 自动分析（双线并行）
  - 线路 A：案情档案 — 合同分析、时间线梳理、证据链构建、风险点识别
  - 线路 B：维权策略 — 法律依据匹配、应对方案、策略模拟、文书准备
        ↓
[Step 4] 生成预览，用户确认
  - 展示案情摘要 + 策略摘要
  - 用户可直接确认或修改
        ↓
[Step 5] 写入文件，立即可用
  - 生成 cases/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含 case.md（案情档案）和 strategy.md（维权策略）
  - 包含 meta.json（元数据）
        ↓
[持续] 进化模式
  - 追加新证据 → merge 进案情档案
  - 对话纠正 → 修正策略
  - 模拟对抗 → 练习应对话术
  - 版本自动存档
```

## 四、输入信息规范

### 4.1 基础信息字段

```yaml
company:        公司名称                    # 必填
slug:           目录标识                    # 必填，如 ali-2025 / tencent-2026
join_date:      入职日期                    # 必填
probation_end:  试用期截止日期               # 可选
position:       岗位名称                    # 可选
level:          职级                       # 可选
salary:         合同约定薪资                # 可选
probation_salary_ratio: 试用期薪资比例       # 可选（如 80%）
contract_type:  合同类型                    # 可选（固定期限/无固定期限/完成一定任务）
contract_period: 合同总期限                 # 可选（如 3 年）

# 争议信息
notify_date:    被通知不转正的日期           # 可选
company_reason: 公司给出的理由              # 可选
my_position:    你认为是否合理              # 可选
current_stage:  目前进展                   # 可选
  - 约谈中
  - 已收到书面通知
  - 已被迫离职
  - 已申请仲裁
  - 仲裁进行中
  - 已结案
```

### 4.2 常见公司理由标签

- `不符合录用条件` — 公司最常用，需举证录用条件是什么、为何不符合
- `工作能力不足` — 需提供考核标准和不达标证据
- `团队调整/岗位取消` — 不属于试用期合法解雇理由
- `价值观不匹配` — 模糊理由，难以举证
- `违反公司制度` — 需证明制度已公示且确实违反
- `态度问题` — 主观判断，难以量化
- `绩效不达标` — 需有明确的考核标准和签字确认

### 4.3 证据类型标签

**劳动关系类**
- `劳动合同` / `offer letter` / `入职登记表` / `社保记录`

**工作表现类**
- `工作成果` / `项目文档` / `代码提交记录` / `邮件往来` / `会议纪要`
- `绩效评估` / `OKR/KPI 记录` / `表扬/嘉奖记录`

**考勤类**
- `打卡记录` / `请假记录` / `加班记录` / `出差记录`

**薪资类**
- `工资条` / `银行流水` / `个税记录` / `社保缴纳记录`

**争议证据类**
- `HR 约谈记录` / `领导谈话截图` / `群聊消息` / `录音转写`
- `解雇通知` / `离职证明` / `劝退微信聊天`

## 六、内置法律知识库

在 `prompts/legal_knowledge.md` 中内置核心法条摘要，确保生成内容的准确性。

### 6.1 核心法条清单

**劳动合同法**
- 第 19 条：试用期期限规定（合同期 1-3 月不得超 1 月，1-3 年不得超 2 月，3 年以上/无固定不得超 6 月）
- 第 20 条：试用期工资不得低于转正工资 80% 或同岗位最低档
- 第 21 条：试用期解除劳动合同的限制
- 第 39 条：用人单位单方解除劳动合同的条件（含第 1 款"在试用期间被证明不符合录用条件"）
- 第 40 条：额外情形下的解除
- 第 46 条：经济补偿金适用情形
- 第 47 条：经济补偿金计算标准
- 第 48 条：违法解除的赔偿金（2N）
- 第 87 条：违法解除的法律后果

**劳动法**
- 第 25 条：试用期解除条件
- 第 28 条：用人单位应支付经济补偿的情形
- 第 77-84 条：劳动争议处理（协商、调解、仲裁、诉讼）

**相关司法解释**
- 《最高人民法院关于审理劳动争议案件适用法律问题的解释》核心条款
- 举证责任分配原则（用人单位对解除合法性负举证责任）

### 6.2 常见维权场景判定

| 场景 | 合法性 | 维权方向 |
|------|--------|---------|
| 试用期内无理由辞退 | ❌ 违法 | 要求继续履行或 2N 赔偿 |
| 以"不符合录用条件"辞退但无录用条件标准 | ❌ 违法 | 指出举证责任在公司 |
| 有录用条件标准，但未提前告知 | ❌ 违法 | 主张不知情 |
| 有录用条件 + 考核不达标 + 已告知 | ⚠️ 可能合法 | 需审查考核过程是否公正 |
| 公司以"团队调整"辞退 | ❌ 不属于试用期解雇理由 | 主张违法解除 |
| 试用期超过法定时长 | ❌ 违法 | 可主张超期部分按转正工资补差 |
| 试用期工资低于 80% | ❌ 违法 | 可主张补足差额 |
| 未签劳动合同 | ❌ 违法 | 可主张双倍工资（入职满 1 月后） |
| 公司强制签署"自愿离职" | ❌ 可撤销 | 收集胁迫证据，主张非自愿 |

### 6.3 仲裁流程指引

```
1. 收集证据（你正在做）
2. 向公司注册地/实际工作地的劳动争议仲裁委员会提交仲裁申请
3. 仲裁委 5 日内决定是否受理
4. 受理后 45 日内作出裁决（复杂案件可延长 15 日）
5. 不服裁决可在 15 日内向法院起诉
6. 仲裁时效：自权利被侵害之日起 1 年
```

## 七、生成内容规范

### 7.1 Part A — 案情档案（case.md）

从文件 + 手动信息中提取该案件的**全部事实基础**。

**提取维度：**

```
① 劳动关系确认
   - 合同关键条款摘要
   - 入职/试用期时间线
   - 薪资约定（试用期 vs 转正）

② 工作表现记录
   - 工作成果清单（项目、文档、代码贡献）
   - 绩效评估结果
   - 获得的正面反馈/表扬
   - 日常工作量证明

③ 考勤记录
   - 出勤情况
   - 加班记录
   - 请假记录

④ 争议事件时间线
   - 按时间排列的关键事件
   - 每个事件对应的证据编号

⑤ 证据清单（结构化）
   - 编号 | 证据名称 | 类型 | 来源 | 证明目的 | 强度评级

⑥ 风险点识别
   - 对你不利的证据或事实
   - 公司可能反驳的角度
   - 需要补充收集的证据
```

⑥ 情绪时间线
   - 按时间记录你的心理状态变化（第一次约谈的震惊/被施压的焦虑/被孤立的无力感）
   - 标注公司的不当施压行为（精神胁迫、冷暴力、孤立排挤等）
   - 在仲裁中可用于证明公司的不当行为模式

**生成结果：** `case.md`

### 7.2 Part B — 维权策略（strategy.md）

结合案情档案和内置法律知识，生成**针对性维权方案**。

**分层结构：**

```
Layer 0 — 阶段感知与全局方向
  - 当前阶段判定（协商中 / 已收通知 / 已离职 / 仲裁中 / 已结案）
  - 基于阶段的默认推荐策略
  - 用户意愿确认（先问用户想怎么做，再给方案）

Layer 1 — 全局方向决策树
  分支 A：协商离职（默认推荐）
    - 赔偿谈判策略（底线、目标、让步节奏）
    - 离职协议注意事项（竞业、背调、社保衔接）
    - 什么时候该接受、什么时候该拒绝
  分支 B：坚决不离职（用户主动选择）
    - 用现有证据反驳公司理由
    - 跟 HR 谈判的话术（为什么你不符合"不符合录用条件"）
    - 即使被强行解除，如何准备恢复劳动关系
  分支 C：已被动离职（事实状态）
    - 违法解除认定 + 2N 赔偿主张
    - 恢复劳动关系仲裁（如用户要求）
    - 赔偿金计算和仲裁申请

Layer 2 — 核心法律判断
  - 公司行为的合法性分析
  - 你最有力的法律依据
  - 预估胜诉可能性（高/中/低 + 理由）

Layer 3 — 证据策略
  - 现有证据充分性评估
  - 需要补充的证据清单
  - 证据链完善建议

Layer 4 — 行动方案
  - 短期（当前阶段应该做的事）
  - 中期（下一阶段准备）
  - 长期（仲裁/诉讼准备，如需要）
  - 关键时间节点提醒

Layer 5 — 应对策略
  - 预测公司/HR 可能的话术和策略
  - 针对每种话术的应对建议
  - 红线提醒（绝对不要签/说的事情）

Layer 6 — 文书模板
  - 劳动仲裁申请书（草稿）
  - 证据清单（结构化）
  - 相关法律条文引用清单

Layer 7 — Correction 层（对话纠正追加）
  - 每条 correction 记录：场景 + 错误判断 + 正确信息
```

**重要原则：策略以用户意愿为核心**
1. **先给全局建议**：基于当前阶段和证据，给出对用户最有利的推荐方向
2. **再问用户想法**：你是否接受这个方向？还是你有别的想法？
3. **按用户选择执行**：用户选择协商，就全力帮协商；用户选择硬刚，就全力帮硬刚
4. **随时可切换**：协商谈崩了，可以无缝切换到仲裁路径

**生成结果：** `strategy.md`

### 7.3 完整组合 SKILL.md

将 `case.md` + `strategy.md` + 内置法律知识合并，生成可直接运行的完整 Skill。

**运行逻辑：**

```
首次调用 /{slug} → 判断当前阶段 → 给出全局方向建议 → 询问用户意愿
用户问维权问题 → 用案情档案的事实 + 法律知识 + 阶段策略 → 给出针对性建议
用户问"公司说 xxx" → 匹配应对策略 → 给出回复话术
用户要模拟对话 → 扮演 HR → 进行对抗演练
用户说"我拿到了新证据" → 进入进化模式 → 更新案情档案 + 重评策略
用户说"我不想离职" / "我想协商" / "我想仲裁" → 切换策略分支
```

## 八、特色功能

### 8.1 对话模拟（对抗演练）

用户可以选择与"虚拟 HR"进行对话练习：

```
用户选择"模拟对抗模式"
    ↓
Skill 扮演公司 HR，使用常见的施压话术
    ↓
用户回复
    ↓
Skill 评估回复是否得当，给出改进建议
    ↓
继续模拟
```

常见 HR 话术库：
- "公司认为你不太适合这个岗位"
- "你还是主动辞职吧，这样不影响你背调"
- "我们给你 N+1 已经很客气了"
- "你仲裁也赢不了的"
- "试用期双方都可以随时解除"
- "你看你考核成绩确实不达标"

### 8.2 法律文书生成

一键生成以下文书草稿：
- 劳动仲裁申请书
- 证据目录
- 法律条文引用清单
- 催告函 / 要求继续履行劳动合同通知

### 8.3 情绪时间线

在案情档案中额外生成一条**情绪时间线**，记录用户的心理状态变化：

```
2026-03-15  😐 正常状态 — 日常工作中，无异常
2026-03-18  😰 首次约谈 — HR 突然通知试用期不通过，感到震惊
2026-03-19  😟 施压升级 — 领导暗示"大家都觉得你不合适"，开始焦虑
2026-03-20  😤 愤怒 — HR 话说"你主动离职对大家都好"，感到被胁迫
2026-03-22  🧘 理性 — 咨询律师后，开始系统收集证据，心态转变
```

作用：
- 记录公司的施压行为模式（可能构成精神胁迫证据）
- 帮助用户复盘自己的状态变化
- 在仲裁中用于证明公司的不当行为模式
- Skill 回复时感知用户情绪状态，调整语气

### 8.4 代价计算器

帮助用户做理性决策，不凭情绪行动：

```
┌─────────────────────────────────────────┐
│           代价计算器                      │
├─────────────┬───────────────────────────┤
│ 方案         │ 协商离职  │ 仲裁维权      │
├─────────────┼──────────┼────────────────┤
│ 预期收益     │ 1个月工资  │ 2个月工资(2N) │
│ 时间成本     │ 1-2周     │ 3-6个月       │
│ 精力成本     │ 低        │ 高            │
│ 背调风险     │ 低        │ 中            │
│ 胜诉概率     │ 100%      │ 70%           │
│ 综合预期值   │ ████████  │ ██████░░      │
└─────────────┴──────────┴────────────────┘
```

计算维度：
- **经济收益**：N / 2N / 补发工资 / 年假折算 / 加班费
- **时间成本**：仲裁周期（45-60天）+ 可能诉讼（3-6个月）
- **机会成本**：影响找下一份工作的时间窗口
- **背调风险**：仲裁记录是否会影响背调（很多公司不查，但部分会）
- **心理成本**：长期对抗的精神压力

**原则**：先给理性分析，再尊重用户选择。如果用户坚定要走仲裁，全力支持。

### 8.5 阶段感知与策略路由

Skill 根据当前阶段自动匹配对应策略，每次调用时先确认阶段：

```
┌──────────────────────────────────────────────┐
│            阶段策略路由                        │
├──────┬───────────────┬───────────────────────┤
│ 阶段  │ 策略重心       │ 关键动作              │
├──────┼───────────────┼───────────────────────┤
│ 协商中 │ 争取最优离职条件│ 谈判话术 + 底线设定    │
│       │               │ 证据收集（暗中）       │
├──────┼───────────────┼───────────────────────┤
│ 被施压 │ 保护自己不签字 │ 红线提醒 + 录音建议    │
│       │ 继续收集证据   │ 每次谈话后的记录模板    │
├──────┼───────────────┼───────────────────────┤
│ 已通知 │ 评估合法性     │ 书面回复模板 +         │
│       │ 准备反击材料   │ 证据保全               │
├──────┼───────────────┼───────────────────────┤
│ 已离职 │ 仲裁准备      │ 仲裁申请书 +           │
│       │               │ 证据清单 + 时效提醒     │
├──────┼───────────────┼───────────────────────┤
│ 仲裁中 │ 庭审准备      │ 陈述词 + 答辩策略 +    │
│       │               │ 对方可能论点预判        │
├──────┼───────────────┼───────────────────────┤
│ 已结案 │ 执行/复盘     │ 申请强制执行 +         │
│       │               │ 经验沉淀               │
└──────┴───────────────┴───────────────────────┘
```

**用户意愿优先原则**（每次策略建议时的决策流程）：

```
1. 先给全局推荐
   "基于你目前的情况，我建议优先协商离职，原因：
    - 你还在试用期内，公司举证门槛相对低
    - 协商可以拿 1 个月赔偿，时间成本几乎为零
    - 仲裁走下来 3-6 个月，预期 2N 但有风险"

2. 问用户想法
   "但这是你的决定。你想怎么做？
    A. 接受协商，帮我把赔偿谈高一点
    B. 我不想走，想留下来继续工作
    C. 我要硬刚，走仲裁
    D. 我还没想好，帮我分析利弊"

3. 按用户选择执行
   A → 谈判策略模式（底线、让步节奏、话术）
   B → 反驳策略模式（用证据证明不符合辞退条件）
   C → 仲裁准备模式（文书、证据链、法律依据）
   D → 代价计算器 + 各路径对比分析

4. 随时可切换
   "协商谈崩了" → 无缝切换到 C（仲裁准备）
   "公司态度软了" → 无缝切换回 A（协商）
```

## 九、进化机制

### 9.1 追加证据进化

```
用户: 我拿到了新的绩效评估截图
        ↓
系统分析新证据
        ↓
判断影响：
  - 增强了证据链 → merge 进 case.md
  - 影响了策略判断 → 更新 strategy.md
  - 两者都有 → 分别更新
        ↓
保存新版本，提示用户变更摘要
```

### 9.2 对话纠正进化

```
用户: "不对，我的试用期是 6 个月不是 3 个月"
用户: "公司还有一条理由没说到"
        ↓
系统识别 correction
        ↓
更新 case.md 或 strategy.md
        ↓
重新评估整体策略
```

### 9.3 版本管理

- 每次更新自动存档到 `versions/`
- 支持 `/dagong-rollback {slug} {version}` 回滚
- 保留最近 10 个版本

## 十二、安全边界

1. **仅供参考，不构成法律建议** — 明确告知用户 Skill 输出不能替代律师意见
2. **不鼓励恶意仲裁** — 只帮助合法权益确实受到侵害的人
3. **隐私保护** — 所有数据仅本地存储，不上传任何服务器
4. **情绪支持** — 被劝退是有压力的，Skill 应保持理性和支持性的语气
5. **不做虚假证据** — 只分析和组织真实材料，不帮助伪造证据

## 十四、验证方案

1. **创建流程测试**：运行 `/create-dagong-soul`，手动输入一个模拟案例，验证能否正常生成完整的案件 Skill
2. **证据导入测试**：上传微信截图、邮件 .eml、考勤截图，验证解析是否正常
3. **咨询质量测试**：用生成的案件 Skill 进行多轮问答，验证回答是否准确引用法条、是否贴合个人案情
4. **对抗模拟测试**：启动模拟 HR 模式，验证对话是否逼真、建议是否实用
5. **文书生成测试**：生成仲裁申请书草稿，检查格式和内容是否完整
6. **多公司测试**：创建两个不同公司的案件，验证互不干扰、可独立迭代
