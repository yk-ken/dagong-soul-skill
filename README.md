## 功能特性

| # | 功能 | 说明 |
|---|------|------|
| 1 | **案情档案** | 自动从材料中提取劳动关系、工作表现、证据清单、时间线 |
| 2 | **维权策略** | 基于法律知识生成针对性维权方案（协商 / 不离职 / 仲裁三路分支） |
| 3 | **阶段感知** | 根据当前进展（约谈中 → 被施压 → 已通知 → 已离职 → 仲裁中 → 已结案）自动匹配策略 |
| 4 | **HR 对抗模拟** | 与虚拟 HR 练习谈判，常见话术拆解 + 应对建议 |
| 5 | **代价计算器** | 协商 vs 仲裁的量化对比（收益 / 时间 / 风险 / 心理成本） |
| 6 | **情绪时间线** | 记录心理状态变化 + 公司施压行为模式 |
| 7 | **法律文书生成** | 一键生成仲裁申请书、证据目录、法条引用清单 |
| 8 | **多公司支持** | 每家公司独立案件，互不干扰 |

### Claude Code

> **重要**：Claude Code 从 **git 仓库根目录** 的 `.claude/skills/` 查找 skill。请在正确的位置执行。

```bash

mkdir -p .claude/skills
```

```bash
pip3 install -r requirements.txt
```

## 使用

在 Claude Code 中输入：

```
/create-dagong-soul
```

按提示输入公司名称、入职日期、目前进展，然后选择上传材料或直接口述。所有字段均可跳过，后续可追加。

完成后用 `/{slug}` 调用该案件 Skill。

### 管理命令

| 命令 | 说明 |
|------|------|
| `/create-dagong-soul` | 创建新的维权案件 |
| `/list-cases` | 列出所有案件 |
| `/{slug}` | 查看案情档案 + 维权策略 |
| `/{slug}-sim` | HR 对抗模拟 |
| `/{slug}-docs` | 生成法律文书 |
| `/{slug}-cost` | 代价计算器 |
| `/case-rollback {slug} {version}` | 回滚到历史版本 |
| `/delete-case {slug}` | 删除案件 |

## 效果示例

> 输入：`某科技公司，2025年1月入职，试用期3个月，3月底被约谈说不符合录用条件`

### 场景一：首次咨询

```
用户            ❯ 我被通知试用期不通过了，公司说不符合录用条件，让我主动提离职

打工魂·不灭 ❯ 先别慌，也别急着签字。我帮你理一下现状：

               根据《劳动合同法》第 39 条，用人单位以"不符合录用条件"为由
               在试用期解除劳动合同，需要同时满足三个前提：

               1. 录用条件在入职时已经明确告知你（有书面记录）
               2. 公司有客观证据证明你不符合这些条件
               3. 解除决定在试用期内做出

               请问你入职时有没有签过「试用期录用条件确认书」或类似文件？

用户            ❯ 没有，就签了一份劳动合同，里面写了试用期3个月

打工魂·不灭 ❯ 这是第一个对你有利的点。

               没有明确的录用条件书面确认，公司引用"不符合录用条件"解除
               在仲裁时很难被支持。HR 说"不符合"但拿不出你签字确认的标准，
               这属于举证不能。

               接下来几个关键问题：
               - 约谈时有几个人在场？有没有录音？
               - 他们有没有给你书面通知？
               - 你的工资结构和社保缴纳情况如何？

用户            ❯ 约谈时HR和直属领导两个人，我没录音。也没给书面通知，就是口头说的

打工魂·不灭 ❯ 记住以下建议，按优先级执行：

               【立即行动】
               1. 现在开始所有对话尽量用文字沟通（微信/钉钉），避免纯口头
               2. 如果再次约谈，提前打开手机录音（放在口袋里即可）
               3. 不要签署任何"自愿离职"或"协商解除"的文件

               【证据保全】
               4. 截图保存你的考勤记录、工作成果提交记录
               5. 导出与领导/HR 的聊天记录
               6. 收集能证明你工作表现的材料（周报、交付物确认等）

               你现在可以把已有材料上传给我，我来帮你做全面的证据评估。
```

### 场景二：HR 对抗模拟

```
用户            ❯ 启动HR对抗模拟

打工魂·不灭 ❯ HR 对抗模拟已启动。我将扮演你的 HR，基于常见施压话术进行对话。
               你练习应对，我会在每轮结束后给出评估和建议。
               输入「结束模拟」退出。

               -------

               🏢 HR（模拟）：
               "小王啊，坐。今天找你聊聊，主要是试用期评估的事情。
               你也知道，咱们团队对你的表现有些顾虑，Leader 反馈说你跟预期
               还是有差距的。公司这边也是给你一个机会，希望你主动提离职，
               这样对你后续找工作也好看一些，不会留下被辞退的记录。"

用户            ❯ 我觉得我的工作表现是达标的，能不能具体说说我哪里不符合？

打工魂·不灭 ❯ 📊 应对评估：优秀

               你这一手反问非常好。理由：
               - 把举证责任推回给公司（公司说不符合，那具体是什么？）
               - 没有情绪化，保持冷静
               - 暗示你不接受模糊的"不符合"说法

               💡 注意事项：
               - 如果 HR 给不出具体例子，说明"不符合录用条件"缺乏依据
               - 如果 HR 拿出绩效考核表，注意是否是你签字确认过的
               - HR 可能会转移话题到"公司也是为你好"，不要被带节奏

               -------

               🏢 HR（模拟）：
               "具体的嘛…… Leader 说你上手比较慢，跟团队融合也不太好。
               不过这些不是重点，重点是公司愿意给你一个体面的离开方式。
               你想啊，如果写'试用期被辞退'，下一家公司背调的时候多难看。
               我们写'个人原因'，你好我好大家好。"

用户            ❯ 我需要考虑一下，不能现在答复你

打工魂·不灭 ❯ 📊 应对评估：良好

               没有当场拒绝也没有答应，这是基本正确的策略。"考虑一下"
               给你争取了时间。

               💡 改进建议：
               - 可以更加明确："我会仔细考虑，有结果通知你"
               - 避免说"考虑"但语气犹豫，HR 会认为你在动摇
               - 离开约谈后立即文字记录约谈内容（时间、地点、在场人、说了什么）

               关于 HR 的"背调不好看"话术，这里拆解一下：
               - 这是典型的施压策略，利用信息不对称制造恐惧
               - 事实上，离职原因在背调中并非关键因素
               - "主动离职"意味着你放弃所有维权权利，损失远大于所谓的"不好看"
```

### 场景三：证据追加

```
用户            ❯ 我拿到了试用期间的绩效评估截图，Leader 给我打的是B（符合预期）

打工魂·不灭 ❯ 这是一份非常关键的证据。让我分析一下影响：

               📋 证据评估：

               【直接影响】
               - "B（符合预期）"直接 contradict 公司"不符合录用条件"的说法
               - 如果这个绩效评估是公司官方体系产出的，仲裁时证明力很强
               - 时间线需要确认：这个评估是在约谈之前还是之后？

               【策略影响】
               1. 如果评估在约谈之前 → 公司明知你符合预期仍解除，属于违法解除
                  → 可主张赔偿金（2N，即 2 个月工资）
               2. 如果评估在约谈之后 → 可能是公司事后补做，需要审查评估流程的
                  合规性（是否有你签字、评估标准是否提前公示）

               【建议追加】
               - 确认该绩效评估是否有 Leader 签字或系统截图
               - 如果有邮件通知，保存完整邮件（含发件人、时间、头信息）
               - 截图时注意包含系统界面的时间戳和 URL

               要不要把截图发给我，我帮你更新案情档案和维权策略？

用户            ❯ 发给你了。另外我还有 Leader 在钉钉群里夸我的聊天记录

打工魂·不灭 ❯ 太好了，两份证据形成互补：

               1. 正式绩效评估（B）→ 证明公司官方评估你是合格的
               2. Leader 钉钉夸赞记录 → 证明直属领导的主观评价也是正面的

               合在一起，公司在仲裁中几乎不可能证明"不符合录用条件"。

               我已经更新了你的案件档案：
               - 证据清单新增 2 项，当前共 X 项证据
               - 维权策略从"协商优先"调整为"仲裁有较大胜算"
               - 预估赔偿金额：X 元（N+1 或 2N 方案）

               下一步建议：
               如果公司还没有发正式的书面解除通知，你可以选择——
               A. 继续正常上班，等公司发书面通知（证据更充分）
               B. 主动与 HR 谈判，拿评估结果作为筹码

               你倾向哪个方向？
```

## 生成的 Skill 结构

每个案件生成以下文件：

```
cases/{slug}/
├── SKILL.md                 # Skill 入口（官方 frontmatter）
├── case.md                  # 案情档案（劳动关系、时间线、证据清单）
├── strategy.md              # 维权策略（协商 / 不离职 / 仲裁三路分支）
├── emotion_timeline.md      # 情绪时间线（心理状态 + 公司施压行为）
├── meta.json                # 元数据（公司名称、阶段、版本号）
├── evidence/                # 原始证据文件（聊天记录、截图、文档）
└── versions/                # 历史版本存档（支持回滚）
```

运行逻辑：`收到问题 → 感知当前阶段 → 匹配维权策略 → 调用法律知识 + 证据库 → 输出建议`

## 安全边界

1. **仅供参考**：本工具提供的法律分析和策略建议不构成正式法律意见，不替代专业律师咨询
2. **不鼓励恶意仲裁**：仅在合法权益受到侵害时使用，不支持无事实依据的恶意维权
3. **隐私保护**：所有数据仅存储在本地，不上传任何云端。案件数据支持加密存储
4. **情绪支持**：试用期被辞退是常见的职场经历，不是你的错。如果感到持续焦虑或抑郁，建议寻求心理咨询
5. **不做虚假证据**：本工具仅帮助整理和分析已有材料，绝不协助编造或篡改证据

---

## 注意事项

- **原材料质量决定策略质量**：聊天记录 + 工作文档 > 仅口述
- 建议优先收集：**解雇通知 / 约谈记录** > **劳动合同** > **工作成果** > **考勤记录**
- 仲裁时效 1 年，证据越早收集越好
- 本项目不替代律师，重大争议建议咨询专业律师
